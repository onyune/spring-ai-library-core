## 1. 개요

이 문서에서는 **두 개의 Function을 조합**하여, 도서 검색과 리뷰 조회를 한 번에 처리하는 방법을 배웁니다.

**학습 목표**:
- 여러 Function 정의 방법
- 함수 간 데이터 전달 이해
- AI가 함수를 순차적으로 호출하는 원리
- 함수 체이닝(Chaining) 패턴 이해

**구현할 것**:
- 리뷰 조회 Function (ReviewFunction)
- 검색 + 리뷰 조합 (AiConfig에 2개 Function 등록)

## 2. 복수 Function 조합이란?

### 2.1. 단일 Function vs 복수 Function

**단일 Function (Level 1)**:
```
사용자: "자바 책 찾아줘"
   ↓
[Telegram Bot]
   ↓
[AI] searchBooks() 호출
   ↓
[Telegram] "3권을 찾았습니다..."
```

**복수 Function (이 문서)**:
```
사용자: "자바의 정석 책 정보랑 리뷰도 알려줘"
   ↓
[Telegram Bot]
   ↓
[AI] 순차적 함수 선택
   ↓
[1단계] searchBooks("자바의 정석") → bookId: 123 확보
   ↓
[2단계] getReviews(123) → 리뷰 요약 확보
   ↓
[AI] 종합 답변 생성
   ↓
[Telegram] "자바의 정석을 찾았습니다!
 리뷰 평점은 4.7/5.0이에요..."
```

### 2.2. 함수 간 데이터 전달

**핵심 개념**: 첫 번째 함수의 출력이 두 번째 함수의 입력이 됨

```
searchBooks("자바의 정석")
  ↓ 결과: { bookId: 123, title: "자바의 정석" }
  ↓
getReviews(123)  ← AI가 bookId를 자동 추출하여 호출
```

## 3. ReviewFunction 구현

### 3.1. ReviewResult DTO

**ReviewResult.java** (`ai/function/dto/`):

```java
package com.nhnacademy.library.ai.function.dto;

import lombok.Builder;

import java.util.List;

/**
 * 리뷰 조회 결과
 *
 * @param bookId 도서 ID
 * @param summary 리뷰 요약
 * @param rating 평균 평점
 * @param reviewCount 리뷰 개수
 * @param reviews 개별 리뷰 (상위 N개)
 */
@Builder
public record ReviewResult(
    long bookId,
    String summary,
    double rating,
    int reviewCount,
    List<ReviewItem> reviews
) {
    /**
     * 개별 리뷰 항목
     *
     * @param content 리뷰 내용
     * @param rating 평점
     */
    @Builder
    public record ReviewItem(
        String content,
        int rating
    ) {}

    public static ReviewResult of(long bookId, String summary, double rating,
                                  int reviewCount, List<ReviewItem> reviews) {
        return new ReviewResult(bookId, summary, rating, reviewCount, reviews);
    }

    public static ReviewResult empty(long bookId) {
        return new ReviewResult(
            bookId,
            "리뷰가 없습니다.",
            0.0,
            0,
            List.of()
        );
    }
}
```

### 3.2. ReviewFunction 구현

**ReviewFunction.java** (`ai/function/`):

```java
package com.nhnacademy.library.ai.function;

import com.nhnacademy.library.ai.function.dto.ReviewResult;
import com.nhnacademy.library.core.review.dto.ReviewResponse;
import com.nhnacademy.library.core.review.service.ReviewService;
import com.nhnacademy.library.core.book.service.search.BookSearchService;
import lombok.RequiredArgsConstructor;
import lombok.extern.slf4j.Slf4j;
import org.springframework.data.domain.PageRequest;
import org.springframework.stereotype.Component;

import java.util.List;

/**
 * 리뷰 조회 Function (Spring AI 1.1.2 FunctionToolCallback 방식)
 * <p>
 * AI Agent가 도서의 리뷰를 조회할 때 호출하는 함수입니다.
 * 도서 ID를 받아 리뷰 요약, 평점, 주요 리뷰를 반환합니다.
 * </p>
 */
@Component
@RequiredArgsConstructor
@Slf4j
public class ReviewFunction {

    private final ReviewService reviewService;
    private final BookSearchService bookSearchService;

    private static final int REVIEW_LIMIT = 5;

    /**
     * 리뷰 조회 실행 메서드 (외부에서 호출)
     */
    public ReviewResult getReviews(Long bookId) {
        log.info("[AI Function] 리뷰 조회 시작: bookId={}", bookId);

        try {
            // 1. 리뷰 조회 (상위 N개)
            var reviews = reviewService.getReviews(
                bookId,
                PageRequest.of(0, REVIEW_LIMIT)
            );

            // 2. 리뷰 요약 조회
            String summary = bookSearchService.getReviewSummary(bookId);

            // 3. 평균 평점 계산
            double avgRating = reviews.getContent().stream()
                .mapToInt(ReviewResponse::rating)
                .average()
                .orElse(0.0);

            // 4. 결과 변환
            List<ReviewResult.ReviewItem> reviewItems = reviews.getContent().stream()
                .map(review -> ReviewResult.ReviewItem.of(
                    review.content(),
                    review.rating()
                ))
                .toList();

            log.info("[AI Function] 리뷰 조회 완료: bookId={}, reviewCount={}",
                bookId, reviewItems.size());

            // 5. 결과 반환
            return ReviewResult.of(
                bookId,
                summary,
                avgRating,
                (int) reviews.getTotalElements(),
                reviewItems
            );

        } catch (Exception e) {
            log.error("[AI Function] 리뷰 조회 실패: bookId={}", bookId, e);
            return ReviewResult.empty(bookId);
        }
    }
}
```

## 4. ChatClient에 여러 Function 등록

### 4.1. AiLibraryAssistantService 수정

**AiLibraryAssistantService.java** (`ai/service/AiLibraryAssistantService.java`):

```java
package com.nhnacademy.library.ai.service;

import com.nhnacademy.library.ai.function.BookSearchFunction;
import com.nhnacademy.library.ai.function.LoanCheckFunction;
import com.nhnacademy.library.ai.function.ReviewFunction;
import com.nhnacademy.library.ai.function.dto.*;
import lombok.RequiredArgsConstructor;
import lombok.extern.slf4j.Slf4j;
import org.springframework.ai.chat.client.ChatClient;
import org.springframework.ai.chat.model.ChatModel;
import org.springframework.ai.tool.function.FunctionToolCallback;
import org.springframework.stereotype.Service;

/**
 * Library Assistant Agent 메인 서비스
 * <p>
 * AI Agent와 사용자 간의 상호작용을 담당합니다.
 * Function Calling을 통해 AI가 도서 검색, 리뷰 조회, 대출 가능 확인을 수행합니다.
 * </p>
 */
@Service
@RequiredArgsConstructor
@Slf4j
public class AiLibraryAssistantService {

    private final ChatModel chatModel;
    private final BookSearchFunction bookSearchFunction;
    private final ReviewFunction reviewFunction;
    private final LoanCheckFunction loanCheckFunction;

    public String ask(String userMessage) {
        log.info("[AI Service] 사용자 요청: {}", userMessage);

        // FunctionToolCallback 프로그래밍 방식 생성 (순환 의존성 방지)
        FunctionToolCallback<BookSearchRequest, BookSearchResult> bookSearchTool =
            FunctionToolCallback.builder("BookSearch", bookSearchFunction::searchBooks)
                .description("도서관 시스템에서 도서를 검색합니다. " +
                            "제목, 저자명으로 검색할 수 있으며, 검색어를 받아 도서 목록을 반환합니다.")
                .inputType(BookSearchRequest.class)
                .build();

        FunctionToolCallback<Long, ReviewResult> reviewTool =
            FunctionToolCallback.builder("GetReviews", reviewFunction::getReviews)
                .description("도서의 리뷰 목록을 조회합니다. " +
                            "도서 ID를 받아 리뷰 요약, 평점, 주요 리뷰를 제공합니다.")
                .inputType(Long.class)
                .build();

        FunctionToolCallback<LoanCheckRequest, LoanCheckResult> loanCheckTool =
            FunctionToolCallback.builder("CheckLoanAvailability", loanCheckFunction::checkLoanAvailability)
                .description("도서의 대출 가능 여부를 조회합니다. " +
                            "ISBN을 받아 전국 도서관의 대출 가능 현황을 제공합니다. " +
                            "지역, 위도, 경도를 제공하면 해당 지역의 가까운 도서관을 우선으로 보여줍니다.")
                .inputType(LoanCheckRequest.class)
                .build();

        // ChatClient 생성 (여러 Function Tool 등록)
        ChatClient chatClient = ChatClient.builder(chatModel)
            .defaultToolCallbacks(bookSearchTool, reviewTool, loanCheckTool)
            .build();

        // System Prompt
        String systemPrompt = """
            당신은 도서관 도우미 AI입니다.

            **중요: 반드시 한국어로만 답변하세요.**

            사용자의 요청을 처리할 때:
            1. 필요한 도구(함수)를 호출하여 정보를 수집하세요
            2. 도구 호출 결과를 받으면 반드시 자연스러운 한국어로 답변하세요
            3. 검색 결과, 리뷰, 대출 가능 정보를 종합해서 친절하게 설명하세요
            """;

        String response = chatClient.prompt()
            .system(systemPrompt)
            .user(userMessage)
            .call()
            .content();

        log.info("[AI Service] AI 응답 생성 완료");
        return response;
    }
}
```

### 4.2. 등록 순서

등록된 함수들은 AI가 자동으로 선택됩니다:

```java
.defaultToolCallbacks(
    bookSearchTool,   // 1번째: BookSearch
    reviewTool,       // 2번째: GetReviews
    loanCheckTool     // 3번째: CheckLoanAvailability
)
```

**중요**: AiConfig에 Function을 등록하는 것이 아니라, **AiLibraryAssistantService**에서 매번 ChatClient를 생성할 때 FunctionToolCallback을 등록합니다.

## 7. 함수 체이닝 동작 상세 분석

### 7.1. AI 내부 사고 과정

```
[Step 1] 사용자 입력 분석
사용자: "자바의 정석 책 정보랑 리뷰도 알려줘"
   ↓
[Step 2] 의도 파악
- 키워드 추출: "자바의 정석"
- 요구 파악: "책 정보" + "리뷰"
   ↓
[Step 3] 함수 선택
- 도서 검색이 필요함 → searchBooks 선택
- 리뷰 조회가 필요함 → getReviews 선택
   ↓
[Step 4] 실행 순서 결정
- 먼저 도서 검색 (bookId 필요)
- 그 다음 리뷰 조회 (bookId 사용)
   ↓
[Step 5] 함수 실행
1. searchBooks("자바의 정석")
   → 결과: {bookId: 123, ...}
2. getReviews(123)
   → 결과: {summary: "...", rating: 4.7}
   ↓
[Step 6] 답변 생성
```

### 7.2. 데이터 전달 과정

```
searchBooks 함수 실행
    ↓
BookSearchResult {
    count: 1,
    books: [
        {
            id: 123,
            title: "자바의 정석",
            ...
        }
    ]
}
    ↓
AI가 books[0].id (123) 추출
    ↓
getReviews 함수 실행 (123 전달)
    ↓
ReviewResult {
    bookId: 123,
    summary: "...",
    rating: 4.7
}
```

## 8. 에러 처리

### 8.1. 함수 호출 실패

**시나리오**: 존재하지 않는 도서 ID로 리뷰 조회

**AI 동작**:
```
1. searchBooks("없는책") → 결과 없음
2. AI: "도서를 못 찾았으니 리뷰도 없겠지" 판단
3. getReviews 호출하지 않음
```

### 8.2. 예외 처리

**ReviewFunction**에서 이미 예외 처리를 구현:
```java
try {
    // 리뷰 조회 로직
} catch (Exception e) {
    log.error("[Function] 리뷰 조회 실패", e);
    return ReviewResult.builder()
        .bookId(bookId)
        .summary("리뷰가 없습니다.")
        .rating(0.0)
        .reviewCount(0)
        .reviews(java.util.List.of())
        .build();
}
```
