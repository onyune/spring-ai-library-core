## 2. Spring AI Function Calling 기초

### 2.1. Function Calling이란?

**기존 방식** (Controller가 직접 처리):
```java
// 개발자가 직접 로직 작성
@GetMapping("/search")
public List<Book> search(@RequestParam String query) {
    return bookService.searchBooks(query);
}
```

**Function Calling 방식** (AI가 자동으로 함수 선택):
```java
// AI가 필요할 때 자동으로 호출
@Description("도서를 검색합니다")
public Function<BookSearchRequest, BookSearchResult> searchBooks() {
    return request -> {
        return bookService.searchBooks(request.query());
    };
}
```

### 2.2. 작동 원리 (Telegram Bot 연동)

```
[Step 1] Telegram 사용자 입력
사용자: "자바 책 찾아줘"
   ↓
[Step 2] Telegram Bot 수신
LibraryTelegramBot.onUpdateReceived()
   ↓
[Step 3] AI 서비스에 위임
AiLibraryAssistantService.ask("자바 책 찾아줘")
   ↓
[Step 4] AI 분석
AI: "책 검색이 필요하네"
   ↓
[Step 5] AI 판단
AI: "searchBooks 함수를 호출해야겠다"
   ↓
[Step 6] 함수 실행
searchBooks("자바")
   ↓
[Step 7] 결과 반환
[책1, 책2, 책3]
   ↓
[Step 8] AI 답변 생성
"3권의 자바 책을 찾았습니다..."
   ↓
[Step 9] Telegram으로 응답
사용자에게 메시지 전송
```

### STEP 4: BookSearchFunction 구현

#### 4.1. Function 인터페이스 이해

```java
Function<BookSearchRequest, BookSearchResult>
```

- **입력**: BookSearchRequest (검색어, limit)
- **출력**: BookSearchResult (검색 결과)

#### 4.2. 전체 코드

**BookSearchFunction.java**

```java
package com.nhnacademy.library.ai.function;

import com.nhnacademy.library.ai.function.dto.BookSearchRequest;
import com.nhnacademy.library.ai.function.dto.BookSearchResult;
import com.nhnacademy.library.core.book.domain.SearchType;
import com.nhnacademy.library.core.book.dto.BookSearchResponse;
import com.nhnacademy.library.core.book.service.search.BookSearchService;
import lombok.RequiredArgsConstructor;
import lombok.extern.slf4j.Slf4j;
import org.springframework.data.domain.PageRequest;
import org.springframework.stereotype.Component;

/**
 * 도서 검색 Function (Spring AI 1.1.2 FunctionToolCallback 방식)
 * <p>
 * AI Agent가 도서 검색을 수행할 때 호출하는 함수입니다.
 * 내부 DB를 검색하여 도서 목록을 반환합니다.
 * </p>
 */
@Component
@RequiredArgsConstructor
@Slf4j
public class BookSearchFunction {

    private final BookSearchService bookSearchService;

    private static final int DEFAULT_PAGE_SIZE = 10;

    /**
     * 도서 검색 실행 메서드 (외부에서 호출)
     */
    public BookSearchResult searchBooks(BookSearchRequest request) {
        log.info("[AI Function] 도서 검색 시작: query={}", request.query());

        try {
            // 1. 내부 DB 검색 실행 (하이브리드 검색)
            var coreRequest = new com.nhnacademy.library.core.book.dto.BookSearchRequest(
                request.query(),
                null,
                SearchType.HYBRID,
                null,
                false
            );

            var result = bookSearchService.searchBooks(
                PageRequest.of(0, request.limit()),
                coreRequest
            );

            // 2. 결과 변환
            var bookItems = result.getBooks().getContent().stream()
                .map(this::toBookItem)
                .toList();

            log.info("[AI Function] 검색 완료: {}권", bookItems.size());

            // 3. 결과 반환
            return BookSearchResult.from(bookItems);

        } catch (Exception e) {
            log.error("[AI Function] 검색 실패: query={}", request.query(), e);
            return BookSearchResult.empty();
        }
    }

    /**
     * BookSearchResponse를 BookSearchResult.BookItem으로 변환합니다.
     */
    private BookSearchResult.BookItem toBookItem(BookSearchResponse book) {
        return BookSearchResult.BookItem.from(
            book.getId(),
            book.getTitle(),
            book.getAuthorName(),
            book.getPublisherName(),
            book.getIsbn(),
            "내부 DB"
        );
    }
}
```

### 코드 설명

**1. @Component**
- Spring Bean으로 등록
- AiLibraryAssistantService에서 주입받아 사용 가능

**2. searchBooks() 메서드**
- `Function<BookSearchRequest, BookSearchResult>` 형태의 람다 대신
- 직접 메서드로 구현
- FunctionToolCallback.builder()에서 메서드 레퍼런스로 사용

**3. BookSearchResult.from() / empty()**
- 정적 팩토리 메서드 사용
- 결과 객체 생성을 간소화

**4. 로그**
- `[AI Function]` 접두사로 함수 호출 로깅
- AI가 함수를 호출했는지 확인 가능

```java
package com.nhnacademy.library.core.config;

import org.springframework.ai.chat.client.ChatClient;
import org.springframework.ai.chat.model.ChatModel;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.boot.autoconfigure.condition.ConditionalOnProperty;

@Configuration
public class AiConfig {

    @Bean
    @ConditionalOnProperty(name = "spring.ai.selected-model", havingValue = "ollama")
    public ChatClient qwenChatClient(@Qualifier("ollamaChatModel") ChatModel ollamaChatModel) {
        return ChatClient.builder(ollamaChatModel).build();
    }

    @Bean
    @ConditionalOnProperty(name = "spring.ai.selected-model", havingValue = "ollama")
    public ChatClient llama3ChatClient(@Qualifier("ollamaChatModel") ChatModel ollamaChatModel) {
        return ChatClient.builder(ollamaChatModel).build();
    }

    /**
     * 기본 ChatClient
     * <p>
     * Step 7: qwen2.5를 기본으로 사용 (A2A 오케스트레이션용)
     * </p>
     */
    @Bean
    public ChatClient chatClient(ChatModel chatModel) {
        return ChatClient.builder(chatModel).build();
    }
}
```

### 4.2. 설정 설명

**여러 ChatClient Bean**:
- `qwenChatClient`: AiOrchestrationService용 (A2A 오케스트레이션, Function Calling)
- `llama3ChatClient`: QueryParserAgent용 (한국어 쿼리 파싱)
- `chatClient`: 기본 ChatClient

**중요**: Function은 AiConfig가 아닌 **AiLibraryAssistantService**에서 `FunctionToolCallback`으로 등록합니다.

## 5. AI 서비스 구현

### 5.1. AiLibraryAssistantService

**중요**: Spring AI 1.1.2부터는 **FunctionToolCallback**을 사용하여 Function을 등록합니다.

```java
package com.nhnacademy.library.ai.service;

import com.nhnacademy.library.ai.function.BookSearchFunction;
import com.nhnacademy.library.ai.function.dto.BookSearchRequest;
import com.nhnacademy.library.ai.function.dto.BookSearchResult;
import lombok.RequiredArgsConstructor;
import lombok.extern.slf4j.Slf4j;
import org.springframework.ai.chat.client.ChatClient;
import org.springframework.ai.chat.model.ChatModel;
import org.springframework.ai.tool.function.FunctionToolCallback;
import org.springframework.stereotype.Service;

/**
 * Library Assistant Agent 메인 서비스
 * <p>
 * 사용자의 자연어 질문을 받아 AI가 적절한 함수를 호출하도록 합니다.
 * </p>
 * <p>
 * 순환 의존성을 방지하기 위해 FunctionToolCallback을 Spring Bean으로 등록하지 않고,
 * 서비스 내부에서 프로그래밍 방식으로 생성합니다.
 * </p>
 */
@Service
@RequiredArgsConstructor
@Slf4j
public class AiLibraryAssistantService {

    private final ChatModel chatModel;
    private final BookSearchFunction bookSearchFunction;

    /**
     * 사용자 질문 처리 (AI가 필요한 함수를 자동으로 선택)
     *
     * @param userMessage 사용자 메시지 (예: "자바 책 찾아줘")
     * @return AI 답변
     */
    public String ask(String userMessage) {
        log.info("[AI Service] 사용자 요청: {}", userMessage);

        // FunctionToolCallback 프로그래밍 방식 생성 (순환 의존성 방지)
        FunctionToolCallback<BookSearchRequest, BookSearchResult> bookSearchTool =
            FunctionToolCallback.builder("BookSearch", bookSearchFunction::searchBooks)
                .description("도서관 시스템에서 도서를 검색합니다. " +
                            "제목, 저자명으로 검색할 수 있으며, 검색어를 받아 도서 목록을 반환합니다.")
                .inputType(BookSearchRequest.class)
                .build();

        // ChatClient 생성 (Function Tool 등록)
        ChatClient chatClient = ChatClient.builder(chatModel)
            .defaultToolCallbacks(bookSearchTool)
            .build();

        // System Prompt: AI에게 함수 결과를 바탕으로 자연어 응답하도록 지시
        String systemPrompt = """
            당신은 도서관 도우미 AI입니다.

            **중요: 반드시 한국어로만 답변하세요.**

            사용자의 요청을 처리할 때:
            1. 필요한 도구(함수)를 호출하여 정보를 수집하세요
            2. 도구 호출 결과를 받으면 반드시 자연스러운 한국어로 답변하세요
            """;

        String response = chatClient.prompt()
            .system(systemPrompt)
            .user(userMessage)
            .call()
            .content();

        log.info("[AI Service] AI 응답: {}", response);
        return response;
    }
}
```

### 5.2. FunctionToolCallback 방식 설명

**@Description 어노테이션 vs FunctionToolCallback.builder()**:

| 구方式 | 설명 | Spring AI 버전 |
|--------|------|---------------|
| `@Description` | 함수에 어노테이션을 직접 붙이는 방식 | 1.0.x (구식) |
| `FunctionToolCallback.builder()` | 프로그래밍 방식으로 생성하는 방식 | 1.1.2+ (권장) |

**FunctionToolCallback.builder()의 장점**:
1. 순환 의존성 문제 해결
2. description을 동적으로 설정 가능
3. inputType을 명시적으로 지정 가능
4. 여러 Function을 유연하게 조합 가능

## 6. Telegram Bot 연동

### 6.1. LibraryTelegramBot 수정

**기존 파일**: `external/telegram/bot/LibraryTelegramBot.java`

```java
package com.nhnacademy.library.external.telegram.bot;

import com.nhnacademy.library.ai.service.AiLibraryAssistantService; // Step 7 추가
import com.nhnacademy.library.external.telegram.config.TelegramBotProperties;
import com.nhnacademy.library.external.telegram.handler.CallbackQueryHandler;
import lombok.RequiredArgsConstructor;
import lombok.extern.slf4j.Slf4j;
import org.springframework.stereotype.Service;
import org.telegram.telegrambots.bots.TelegramLongPollingBot;
import org.telegram.telegrambots.meta.api.objects.Update;

/**
 * AI Library Telegram Bot
 *
 * <p>Step 7: Library Assistant Agent 연동</p>
 */
@Slf4j
@RequiredArgsConstructor
public class LibraryTelegramBot extends TelegramLongPollingBot {

    private final TelegramBotProperties properties;
    private final CallbackQueryHandler callbackQueryHandler;

    // Step 7: AI Agent 서비스 추가
    private final AiLibraryAssistantService aiLibraryAssistantService;

    @Override
    public void onUpdateReceived(Update update) {
        try {
            // Callback Query 처리를 최우선으로 수행
            if (update.hasCallbackQuery()) {
                callbackQueryHandler.handleCallback(update);
                return;
            }

            // 일반 메시지 처리
            if (update.hasMessage() && update.getMessage().hasText()) {
                String messageText = update.getMessage().getText();
                Long chatId = update.getMessage().getChatId();

                log.info("[Telegram] Received message from chatId {}: {}", chatId, messageText);

                // Command 분기 처리
                if (messageText.startsWith("/")) {
                    handleCommand(update, messageText);
                } else {
                    // Step 7: AI Agent에게 위임
                    handleAiAgentSearch(update, messageText);
                }
            }
        } catch (Exception e) {
            log.error("[Telegram] Error: {}", e.getMessage(), e);
        }
    }

    /**
     * AI Agent 검색 처리 (Step 7)
     *
     * @param update Telegram Update
     * @param keyword 검색어/사용자 메시지
     */
    private void handleAiAgentSearch(Update update, String keyword) {
        Long chatId = update.getMessage().getChatId();
        log.info("[Telegram] AI Agent search for: {}, chatId: {}", keyword, chatId);

        try {
            // 최근 검색어 저장 (피드백용)
            callbackQueryHandler.setRecentQuery(chatId, keyword);

            // Step 7: AI Agent에게 위임
            String aiResponse = aiLibraryAssistantService.ask(keyword);

            // AI 응답을 Telegram으로 전송
            sendSimpleMessage(chatId, aiResponse);

        } catch (Exception e) {
            log.error("[Telegram] AI Agent search failed: {}", e.getMessage(), e);
            sendSimpleMessage(chatId, "검색 중 오류가 발생했습니다. 다시 시도해주세요.");
        }
    }

    /**
     * Command 처리 (기존 /search 등은 유지)
     */
    private void handleCommand(Update update, String command) {
        Long chatId = update.getMessage().getChatId();

        switch (command) {
            case "/start":
                sendWelcomeMessage(chatId);
                break;
            case "/help":
                sendHelpMessage(chatId);
                break;
            default:
                sendSimpleMessage(chatId, "❌ 알 수 없는 Command입니다.\n\n도움이 필요하시면 /help를 입력하세요.");
                break;
        }
    }

    /**
     * 간단 메시지 전송
     */
    private void sendSimpleMessage(Long chatId, String text) {
        if (text == null || text.isBlank()) {
            return;
        }

        try {
            var message = new org.telegram.telegrambots.meta.api.methods.send.SendMessage(
                chatId.toString(),
                text
            );
            this.execute(message);
        } catch (Exception e) {
            log.error("[Telegram] Failed to send message: {}", e.getMessage());
        }
    }

    private void sendWelcomeMessage(Long chatId) {
        String welcome = """
            🎉 AI Library Bot에 오신 것을 환영합니다!

            이 Bot은 AI 기반 Library Assistant Agent입니다.

            사용법:
            • 도서 제목이나 키워드를 입력하면 자동 검색됩니다
            • 자연어로 대화하듯 검색할 수 있습니다

            예시:
            • 자바 책 찾아줘
            • 해리포터 관련 책 알려줘
            • 주식 투자 추천 도서

            도움이 필요하시면 /help를 입력하세요
            """;
        sendSimpleMessage(chatId, welcome);
    }

    private void sendHelpMessage(Long chatId) {
        String help = """
            📖 도움말

            Command:
            /start - Bot 시작
            /help - 도움말

            검색 예시:
            • 자바 책
            • 해리포터
            • 주식 투자 방법
            • AI 딥러닝 입문

            자연어 검색 예시:
            • 자바의 정석 책 정보 알려줘
            • 해리포터와 비슷한 판타지 책
            • 주식 초보자가 읽기 좋은 책

            💡 팁: 그냥 평소처럼 말하듯 검색어를 입력해보세요!
            """;
        sendSimpleMessage(chatId, help);
    }

    @Override
    public String getBotUsername() {
        return properties.getUsername();
    }

    @Override
    public String getBotToken() {
        return properties.getToken();
    }
}
```

## 8. 디버깅 팁

### 8.1. 로그 확인

```log
[Function] 도서 검색 시작: query=자바
[Function] 검색 완료: 5권
```

이 로그가 보이면 AI가 함수를 성공적으로 호출한 것입니다!

### 8.2. AI가 함수를 호출하지 않을 때

**원인 1**: @Description이 불명확함
```java
// 나쁜 예
@Description("책")

// 명확한 예
@Description("도서관 시스템에서 도서를 검색합니다. 제목, 저자명으로 검색할 수 있습니다.")
```

**원인 2**: 사용자 질문이 모호함
```
사용자: "날씨 좋네"
→ AI: 검색 필요 없다고 판단
```

**원인 3**: 파라미터 타입 불일치
```java
// Request: int limit
// AI가 문자열을 보내면 에러 발생 가능
```

### 8.3. Telegram Bot 연동 문제

**문제**: Telegram Bot이 메시지를 받지 못함

**해결**:
1. Bot Token 확인 (`application.properties`)
2. Bot이 사용자에게 메시지를 먼저 보내야 함 ( privacy mode 해제 )
3. 웹훅 대신 폴링 방식 사용 중인지 확인
