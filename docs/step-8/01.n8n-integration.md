## 1. 개요

n8n(node-n8n)은 **시각적 워크플로우 자동화 도구**입니다. 이 보너스 미션에서는 Spring AI Function Calling 시스템을 n8n과 연동하여 복잡한 워크플로우를 시각화하고 확장하는 방법을 배웁니다.

### n8n이란?

```
코드 없는 워크플로우 자동화 도구

[특징]
- 400+ 서비스 통합 (Slack, Google Sheets, GitHub 등)
- 시각적 노드 기반 워크플로우 설계
- 오픈 소스 (자체 호스팅 가능)
- TypeScript/JavaScript로 커스텀 노드 개발 가능
```

### 왜 n8n과 Spring AI를 결합하는가?

**Spring AI만 사용할 때:**
```java
// 코드로 함수 순서 정의
searchBooks() → getReviews() → checkLoanAvailability()
// 수정하려면 코드를 다시 컴파일해야 함
```

**n8n과 결합할 때:**
```
[시각적 노드 배치]
도서 검색 → 리뷰 조회 → 대출 확인 → Slack 알림
  ↓           ↓           ↓           ↓
드래그 앤 드롭으로 순서 변경 가능
```

## 3. n8n 설치 및 설정

### 3.1. Docker로 n8n 설치

**docker-compose.yml**:
```yaml
version: '3.8'
services:
  n8n:
    image: n8nio/n8n
    ports:
      - "5678:5678"
    environment:
      - N8N_BASIC_AUTH_ACTIVE=true
      - N8N_BASIC_AUTH_USER=admin
      - N8N_BASIC_AUTH_PASSWORD=password
      - WEBHOOK_URL=http://localhost:5678/
    volumes:
      - n8n_data:/home/node/.n8n
    restart: always

volumes:
  n8n_data:
```

**실행**:
```bash
docker-compose up -d
```

**접속**: http://localhost:5678

### 3.2. 첫 워크플로우 생성

1. **새 워크플로우 클릭**
2. **노드 추가**: `HTTP Request` 노드 선택
3. **Spring API 엔드포인트 설정**:
   - Method: `POST`
   - URL: `http://localhost:8080/api/ai/book-search`
   - Authentication: `None` (개발 환경)

## 4. n8n 워크플로우 구축

### 4.1. 워크플로우 1: 도서 검색 알림

**목표**: 사용자가 도서를 검색하면 결과를 Slack으로 전송

```
[Webhook] → [HTTP Request] → [Slack] → [IF 분기]
   │            │               │           │
   │         검색 API         결과       조건별
   │         호출             전송       처리
```

#### 노드 1: Webhook (트리거)
```json
{
  "path": "book-search",
  "method": "POST",
  "responseMode": "responseNode"
}
```

#### 노드 2: HTTP Request (도서 검색)
```json
{
  "method": "POST",
  "url": "http://localhost:8080/api/ai/book-search",
  "authentication": "none",
  "sendBody": true,
  "bodyParameters": {
    "parameters": [
      {
        "name": "query",
        "value": "={{ $json.query }}"
      }
    ]
  }
}
```

#### 노드 3: Slack (알림 전송)
```json
{
  "channel": "#library-notifications",
  "text": "📚 새로운 도서 검색:\n\n{{ $json.books }}"
}
```

### 4.2. 워크플로우 2: 리뷰 요약 자동화

**목표**: 새 리뷰가 등록되면 AI 요약 생성 후 데이터베이스 저장

```
[Webhook] → [HTTP Request]     → [HTTP Request]  → [PostgreSQL]
   │            │(리뷰 조회)      │(요약 생성)        │
   │          리뷰 10개      AI 요약        저장
   │          가져오기       생성
```

#### 노드 1: Webhook (리뷰 등록 트리거)
```json
{
  "path": "review-added",
  "method": "POST",
  "responseMode": "lastNode"
}
```

#### 노드 2: HTTP Request (리뷰 조회)
```json
{
  "method": "GET",
  "url": "http://localhost:8080/api/reviews/book/{{ $json.bookId }}"
}
```

#### 노드 3: HTTP Request (AI 요약)
```json
{
  "method": "POST",
  "url": "http://localhost:8080/api/ai/summarize-reviews",
  "sendBody": true,
  "bodyParameters": {
    "parameters": [
      {
        "name": "reviews",
        "value": "={{ $json.reviews }}"
      }
    ]
  }
}
```

#### 노드 4: PostgreSQL (요약 저장)
```json
{
  "operation": "update",
  "table": "book_review_summaries",
  "columns": {
    "review_summary": "={{ $json.summary }}"
  }
}
```

### 4.3. 워크플로우 3: 신간 도서 자동 등록 (심화)

## 5. Spring Boot API 엔드포인트

n8n에서 호출할 수 있도록 HTTP 엔드포인트를 노출합니다.

### 5.1. 도서 검색 API

**AiSearchController.java** (기존 파일 수정):
```java
@RestController
@RequestMapping("/api/ai")
@RequiredArgsConstructor
public class AiSearchController {

    private final AiOrchestrationService aiOrchestrationService;

    /**
     * n8n에서 호출할 수 있는 도서 검색 API
     */
    @PostMapping("/book-search")
    public ResponseEntity<Map<String, Object>> searchBooks(
        @RequestBody Map<String, String> request
    ) {
        String query = request.get("query");
        String result = aiOrchestrationService.orchestrate(
            "도서 검색: " + query
        );

        return ResponseEntity.ok(Map.of(
            "query", query,
            "result", result,
            "timestamp", LocalDateTime.now()
        ));
    }
}
```

### 5.2. 리뷰 요약 API

```java
/**
 * n8n에서 호출할 수 있는 리뷰 요약 API
 */
@PostMapping("/summarize-reviews")
public ResponseEntity<Map<String, Object>> summarizeReviews(
    @RequestBody Map<String, List<String>> request
) {
    List<String> reviews = request.get("reviews");
    String summary = reviewSummarizer.summarizeReviews(reviews);

    return ResponseEntity.ok(Map.of(
        "summary", summary,
        "reviewCount", reviews.size()
    ));
}
```

### 5.3. 도서 등록 API

```java
/**
 * n8n에서 호출할 수 있는 도서 등록 API
 */
@PostMapping("/register-book")
public ResponseEntity<Map<String, Object>> registerBook(
    @RequestBody Map<String, String> request
) {
    BookDto book = bookService.register(
        request.get("isbn13"),
        request.get("title"),
        request.get("authors"),
        request.get("publisher")
    );

    return ResponseEntity.ok(Map.of(
        "bookId", book.id(),
        "status", "registered"
    ));
}
```

### 미션 목표

n8n을 사용하여 복잡한 AI 워크플로우를 시각화하고 자동화합니다.

### 시나리오

```
"사용자가 Telegram으로 '자바 책 추천해줘'라고 메시지를 보내면:
1. Spring AI가 도서를 검색
2. 사용자의 피드백 이력을 분석
3. 개인화된 추천 결과 생성
4. n8n 워크플로우가 결과를 Slack으로 전송
5. 동시에 데이터베이스에 검색 로그 저장"
```

### 구현 가이드

#### STEP 1: Spring API 엔드포인트 준비

1. **개인화된 검색 API**:
```java
@PostMapping("/personalized-search")
public ResponseEntity<Map<String, Object>> personalizedSearch(
    @RequestBody Map<String, String> request
) {
    String query = request.get("query");
    Long chatId = Long.valueOf(request.get("chatId"));

    // 개인화된 검색 실행
    BookSearchResult result = bookSearchService.searchBooks(
        pageable, request, chatId
    );

    return ResponseEntity.ok(Map.of(
        "books", result.getBooks(),
        "personalized", true
    ));
}
```

2. **검색 로그 저장 API**:
```java
@PostMapping("/log-search")
public ResponseEntity<Void> logSearch(
    @RequestBody Map<String, Object> request
) {
    searchLogService.save(
        (String) request.get("query"),
        (Long) request.get("chatId"),
        (int) request.get("resultCount")
    );
    return ResponseEntity.ok().build();
}
```

#### STEP 2: n8n 워크플로우 설계

**노드 구성**:
```
[Webhook]          → [HTTP Request]      → [Code]     → [Slack]
   │                    │                  │             │
Telegram 메시지   개인화 검색 API    필터링      알림 전송
수신               호출             (평점 4.0+)
```

**Webhook 노드**:
```json
{
  "path": "telegram-bot",
  "method": "POST",
  "responseMode": "responseNode"
}
```

**HTTP Request 노드**:
```json
{
  "method": "POST",
  "url": "http://localhost:8080/api/ai/personalized-search",
  "sendBody": true,
  "bodyParameters": {
    "parameters": [
      {
        "name": "query",
        "value": "={{ $json.message.text }}"
      },
      {
        "name": "chatId",
        "value": "={{ $json.message.chat.id }}"
      }
    ]
  }
}
```

**Code 노드** (평점 필터링):
```javascript
// 평점 4.0 이상인 도서만 필터링
const books = $input.all()[0].json.books;
const filtered = books.filter(book => book.rating >= 4.0);

return filtered.map(book => ({
  json: book
}));
```

**Slack 노드**:
```json
{
  "channel": "#book-recommendations",
  "text": "📚 추천 도서:\n\n{{ $json.title }}\n평점: {{ $json.rating }}/5.0"
}
```
